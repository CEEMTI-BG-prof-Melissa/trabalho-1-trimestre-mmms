Biblioteca Digital M's

Requisitos Funcionais Requisitos Não Funcionais Requisitos Inversos

Requisitos Funcionais

  • O sistema deve permitir que os alunos visualizem quais livros estão disponíveis.
  • O sistema deve permitir a reserva de livros com antecedência.
  • O sistema deve enviar notificações automáticas para alertar sobre a data de devolução.
  • O sistema deve recomendar livros específicos para as turmas.
  • O sistema deve registrar sugestões de novos títulos.
  • O sistema deve oferecer acesso a livros digitais.
  • O sistema deve prover um espaço para o professor sugerir livros com base no perfil do aluno.
  • O sistema deve enviar alertas para evitar esquecimentos.

Requisitos Não Funcionais

  • O sistema deve ser compatível com dispositivos móveis.
  • O sistema deve reduzir o esforço manual e o deslocamento desnecessário.
  • O sistema deve garantir que apenas alunos e professores da escola tenham acesso (controle de acesso/login).
  • A interface deve ser acessível e incentivar a leitura (fácil de usar)
  • O sistema deve estar disponível para consulta remota (digital), permitindo que o aluno verifique o acervo de qualquer lugar.
  • O sistema deve garantir que o controle de empréstimos e devoluções seja preciso para evitar perdas (extravios).
  • O acesso digital deve eliminar a fila de espera dos livros físicos.
  • O sistema deve responder a consultas de busca de livros em até 2 segundos, mesmo com até 200 usuários simultâneos.
  • O sistema deve exigir autenticação de usuários e garantir que dados sensíveis sejam armazenados com criptografia.
  • O sistema deve estar disponível 24 horas por dia, com no mínimo 99% de uptime mensal e rotinas de backup diário automático.
  • O sistema deve possuir interface intuitiva, permitindo que um usuário realize operações básicas (busca, empréstimo e devolução) em no máximo 3 cliques, sem necessidade de treinamento prévio.

Requisitos Inversos

  • O sistema não deve permitir que pessoas externas à escola (comunidade fora da "Futuro do Saber") acessem os recursos ou o acervo.
  • O sistema não deve permitir a reserva de um livro que já esteja emprestado ou reservado por outro aluno no mesmo período.
  • O sistema não deve permitir que um atraso ocorra sem que o lembrete automático tenha sido disparado (para evitar a desorganização citada).
  • o sistema não deve permitir que o aluno precise ir fisicamente à biblioteca para descobrir que um livro não está lá.
  • O sistema não deve permitir que o aluno perca o prazo por falta de aviso.
